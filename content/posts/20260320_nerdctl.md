1. nerdctl pull 镜像时使用代理
    1. containerd 代理配置

    - root
        /etc/containerd/certs.d/docker.io/hosts.toml
    - rootless
        ~/.config/containerd/certs.d/docker.io/hosts.toml

        ~~~toml
        # 核心：docker.io 镜像重定向到 Harbor 的 /v2/ 接口
        server = "https://xx.xx.xx.xx"

        # 非安全配置（测试环境，生产可删除）
        [host."https://xx.xx.xx.xx"]
        capabilities = ["pull", "resolve"]
        ~~~
    
    2. harbor 中创建一个 library 仓库，代理到 docker.io

2. buildkit

    - rootless ~/.config/buildkit/buildkitd.toml
        ~~~toml
        [registry."docker.io"]
        mirrors = ["https://xx.xx.xx.xx"]

        [registry."xx.xx.xx.xx"]
        http = false
        insecure = false
        ~~~
